# 에이전틱 코딩 가이드

> Claude Code를 활용한 에이전틱 코딩 학습 자료
> AI 코딩 도구 경험 여부와 관계없이 누구나 시작할 수 있는 실전 가이드

---

## 에이전틱 코딩이란?

### AI 코딩 방식의 진화

AI를 활용한 코딩 방식은 네 단계를 거쳐 진화해왔다.

**1단계: 코드 자동완성 (2021~)**

GitHub Copilot이 대표적이다. 개발자가 코드를 작성하면 다음 줄을 예측해서 제안하는 방식이다. 타이핑 속도를 높여주지만, 모든 결정은 여전히 개발자가 내린다.

**2단계: 대화형 코딩 어시스턴트 (2023~)**

ChatGPT, Claude.ai 같은 챗봇에 코드를 붙여넣고 질문하는 방식이다. 문맥 이해력은 올라갔지만, 복사-붙여넣기 과정이 필수이고 실행 환경과 분리되어 있다는 한계가 있다.

**3단계: IDE 통합 AI (2024~)**

Cursor, Windsurf 같은 AI 네이티브 IDE가 등장한다. AI가 프로젝트 파일을 직접 읽고, 에디터 안에서 코드를 수정한다. 복사-붙여넣기 없이 작업할 수 있게 됐지만, 여전히 개발자가 파일 단위로 작업을 지시하는 구조다.

**4단계: 에이전틱 코딩 (2025~)**

Claude Code 등이 여기에 해당한다. AI가 단순히 코드를 제안하는 게 아니라, 스스로 계획을 세우고, 파일을 생성하고, 명령어를 실행하고, 테스트를 돌리고, 실패하면 수정하는 전체 사이클을 자율적으로 수행한다. 더 나아가 여러 에이전트가 역할을 나눠 병렬로 협업하는 멀티 에이전트 방식까지 발전했다.

### 기존 방식과의 차이

기존 AI 코딩 도구는 "자동완성" 혹은 "질의응답" 패러다임이었다. 개발자가 주도권을 쥐고, AI는 보조 역할을 했다.

에이전틱 코딩은 근본적으로 다르다. **AI 에이전트가 자율적으로 코드베이스를 탐색하고, 파일을 생성·수정하며, 명령어를 실행하고, 테스트를 수행한다.** 개발자는 "무엇을 만들 것인가"를 지시하고, 에이전트는 "어떻게 만들 것인가"를 스스로 결정하여 실행한다.

        D --> E[코드 작성]

        F --> G[오류 수정]
        G -->|결과 보고| C
    end
`}</Mermaid>

### 마인드셋 전환

경험 많은 개발자가 에이전틱 코딩을 시작할 때 가장 흔히 겪는 어려움은 **"통제권을 내려놓는 것"**이다.

**"어떻게(How)" 대신 "무엇을(What)" 지시하기**: 에이전트에게 구현 결정권을 줄수록 더 나은 결과를 얻는다.

**코드 리뷰어로 역할 전환**: 직접 코드를 작성하는 대신, 에이전트가 생성한 코드를 리뷰하고 방향을 수정하는 역할에 집중한다. 시니어 개발자가 주니어를 멘토링하는 것과 유사하다.

**반복적 대화로 정밀도 높이기**: 한 번의 완벽한 프롬프트보다 여러 번의 대화를 통해 점진적으로 원하는 결과에 도달하는 것이 효과적이다.

## 실전 팁과 안티패턴

### 효과적인 프롬프트 작성

```
❌ 나쁜 예:
"API를 만들어줘"

✅ 좋은 예:
"태스크 상태를 변경하는 PATCH /api/tasks/{id}/status 엔드포인트를
만들어줘. 요청 본문은 {status: 'todo' | 'in_progress' | 'done'}이고,
프로젝트 멤버만 접근 가능해야 해. 상태 변경 시 updated_at을 자동
갱신하고, 유효하지 않은 상태값에 대해 422 에러를 반환해야 해.
pytest 테스트도 함께 작성해줘."
```

**좋은 프롬프트 체크리스트**:
- 무엇을 만들지 명확한가?
- 입력/출력이 정의되어 있는가?
- 제약 조건과 예외 케이스를 언급했는가?
- 검증 방법(테스트)을 요청했는가?

### 흔한 안티패턴

| 안티패턴 | 문제 | 해결 |
|---------|------|------|
| 과도한 마이크로 관리 | 에이전트에게 한 줄씩 지시하면 비효율적 | 목표와 제약 조건을 주고 자율성을 부여 |
| CLAUDE.md 미작성 | 에이전트가 프로젝트 컨텍스트를 모름 | 프로젝트 시작 시 반드시 CLAUDE.md 작성 |
| 결과 미검증 | 에이전트의 출력을 무비판적으로 수용 | 각 단계마다 테스트 실행 및 코드 리뷰 |
| 단일 거대 프롬프트 | 너무 많은 것을 한 번에 요청 | 작업을 단계별로 나누어 지시 |
| 에이전트 팀 남용 | 단순 작업에 팀 구성 | 병렬 가치가 있을 때만 에이전트 팀 사용 |

### 에이전트와의 효과적인 대화 패턴

**점진적 구체화**: 큰 그림에서 시작하여 세부 사항으로 좁혀간다.

```
1차: "사용자 인증 시스템을 만들어줘"
2차: "비밀번호 해싱에 bcrypt를 사용하고, 토큰 만료 시간은 30분으로 해줘"
3차: "리프레시 토큰도 추가하고, 로그아웃 시 토큰을 블랙리스트에 추가해줘"
```

**명시적 피드백**: 마음에 들지 않는 부분을 구체적으로 지적한다.

```
❌ "다시 해줘"
✅ "서비스 레이어의 에러 처리가 너무 단순해. 각 비즈니스 로직별로
    커스텀 예외를 정의하고, 적절한 HTTP 상태 코드를 매핑해줘."
```

## 빠른 시작

```bash
# Claude Code 설치
npm install -g @anthropic-ai/claude-code

# 설치 확인
claude doctor

# 프로젝트 시작
mkdir my-project && cd my-project
git init
claude
```

자세한 설치 방법은 [Claude Code 설치 가이드](/claude-code/installation)를 참고한다.
실습은 [튜토리얼](/tutorial)에서 TaskFlow 프로젝트를 직접 만들어볼 수 있다.

# Agentic Coding Guide

> **Claude Code를 활용한 에이전틱 코딩 학습 자료**
> AI 코딩 도구 경험 여부와 관계없이 누구나 시작할 수 있는 실전 가이드

---

## 📚 문서 소개

이 저장소는 AI 에이전트를 활용한 에이전틱 코딩(Agentic Coding)을 학습하고 실무에 적용하기 위한 종합 가이드입니다. Claude Code를 중심으로 서브에이전트, 에이전트 팀, 멀티 에이전트 협업 등 최신 AI 개발 방법론을 다룹니다.

### 대상 독자

- AI 코딩 도구 경험이 적거나 없는 개발자
- 에이전틱 코딩을 체계적으로 학습하고 싶은 개발자
- 팀에 에이전틱 코딩을 도입하려는 리더

> **처음이라면?** Claude Code 사용 경험이 없어도 괜찮습니다. [온보딩 핸드북](/getting-started/onboarding-handbook)에서 설치부터 첫 프로젝트 완성까지 단계별로 안내합니다.

### 0. 프로젝트 시작 전 필수 가이드 ⭐

새 프로젝트를 시작하기 전에 [문서 작성 순서 가이드](/project-management/document-creation-order)를 먼저 읽어보세요:

- **왜 필요한가**: 올바른 순서로 문서를 작성하지 않으면 기술 스택 불일치, 불완전한 구현 계획 등의 문제 발생
- **6단계 파이프라인**: PRD → 기술 분석 → CLAUDE.md → 서브에이전트 파일 → 구현 계획 → README.md
- **소요 시간**: 15분 (실습 적용 시 1-2시간)

### 1. Claude Code 설치

```bash
# macOS/Linux
npm install -g @anthropic-ai/claude-code

# 설치 확인
claude doctor
```

### 2. 온보딩 핸드북으로 시작하기

에이전틱 코딩이 처음이라면 [온보딩 핸드북](/getting-started/onboarding-handbook)부터 시작하세요:

```bash
# 튜토리얼 프로젝트 시작
mkdir taskflow && cd taskflow
git init
claude
```

기본 사용법을 익혔다면 [에이전트 개념 가이드](/claude-code/agent-concepts)로 심화 학습:

- 서브에이전트 정의 및 활용
- 에이전트 팀 구성 및 병렬 작업
- Git Worktree 기반 개발 전략

### 4. 고급 프레임워크 탐구

체계적인 개발 방법론이 필요하다면 [MoAI-ADK 가이드](/moai/adk-guide) 참고:

## 🎯 핵심 개념

### 에이전틱 코딩이란?

AI 에이전트가 단순히 코드를 제안하는 것을 넘어, **스스로 계획을 세우고, 파일을 생성하고, 명령어를 실행하고, 테스트를 돌리고, 실패하면 수정하는 전체 개발 사이클을 자율적으로 수행**하는 개발 방식입니다.

### 왜 에이전틱 코딩인가?

```
전통적 개발                     에이전틱 코딩
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
개발자 1명 = 결과물 1명분        개발자 1명 = 결과물 3-5명분
                                (에이전트 오케스트레이션)

코드 작성 → 테스트 → 디버깅     목표 지시 → 에이전트 자율 실행
(모든 단계 수동)                (자동화된 사이클)

순차적 작업                     병렬 협업
                                (멀티 에이전트)
```

### 주요 구성 요소

- **CLAUDE.md**: 프로젝트의 컨텍스트와 규칙을 정의하는 설정 파일
- **서브에이전트**: 특정 역할(백엔드, 프론트엔드, QA 등)을 수행하는 전문 에이전트
- **에이전트 팀**: 여러 에이전트가 병렬로 협업하는 시스템
- **Plan Mode**: 구현 전 계획을 세우고 검토하는 모드

## 💡 실전 활용 예시

### 프롬프트 작성 팁

**❌ 나쁜 예:**
```
API를 만들어줘
```

**✅ 좋은 예:**
```
태스크 상태를 변경하는 PATCH /api/tasks/{id}/status 엔드포인트를 만들어줘.
요청 본문은 {status: 'todo' | 'in_progress' | 'done'}이고,
프로젝트 멤버만 접근 가능해야 해.
pytest 테스트도 함께 작성해줘.
```

### 멀티 에이전트 협업 예시

```
에이전트 팀을 만들어서 다음 작업을 병렬로 진행해줘:

팀 구성:
- 백엔드 개발자: Task 모델에 due_date 추가, Label 모델 생성
- 프론트엔드 개발자: UI 컴포넌트 업데이트 (마감일 표시, 라벨 뱃지)
- QA 엔지니어: 기존 테스트 확인 및 새 기능 테스트 작성

백엔드 개발자가 스키마 변경을 완료하면 프론트엔드 개발자에게 알려줘.
```

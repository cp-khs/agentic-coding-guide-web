## 소개

### 두 가지 협업 방식

이 튜토리얼은 tmux를 활용한 **두 가지 다른 팀 작업 방식**을 다룹니다:

#### 1️⃣ 표준 tmux - 사람들 간의 협업
- 여러 사람이 같은 터미널 세션 공유
- 페어 프로그래밍, 원격 협업
- 사람이 직접 모든 작업 수행

#### 2️⃣ Claude Code Agent Teams - AI 에이전트 팀 협업 🆕
- **AI 에이전트들이 자동으로 작업 수행**
- tmux에서 각 에이전트를 시각적으로 확인
- 에이전트들이 서로 소통하며 병렬 작업
- **"Agent Team을 만들어줘"라고 하면 자동으로 팀 생성!**

### 왜 중요한가?

**표준 tmux**: 
- 🤝 원격 팀원과 실시간 협업
- 👀 같은 화면을 보며 페어 프로그래밍
- 💾 연결이 끊겨도 세션 유지

**Claude Code Agent Teams**:
- 🤖 AI 에이전트들이 병렬로 작업
- 🎯 각 에이전트에게 특정 역할 할당
- ⚡ 대규모 프로젝트를 빠르게 처리
- 👁️ tmux로 모든 에이전트 작업 실시간 모니터링

### 5. tmux 기본 설정 (~/.tmux.conf)

```bash
# 설정 파일 생성
nano ~/.tmux.conf
```

**추천 설정:**

```bash
# 마우스 모드 활성화
set -g mouse on

# 창 번호를 1부터 시작
set -g base-index 1
setw -g pane-base-index 1

# 창 번호 자동 재정렬
set -g renumber-windows on

# 256 컬러 지원
set -g default-terminal "screen-256color"

# 상태바 스타일
set -g status-style 'bg=colour235 fg=colour255'
set -g status-left-length 40
set -g status-left '[#S] '
set -g status-right '%Y-%m-%d %H:%M'

# 분할창 이동을 vim 스타일로
bind h select-pane -L
bind j select-pane -D
bind k select-pane -U
bind l select-pane -R

# 새 창을 현재 경로에서 열기
bind '"' split-window -c "#{pane_current_path}"
bind % split-window -h -c "#{pane_current_path}"
bind c new-window -c "#{pane_current_path}"

# 팬 크기 조절 (Ctrl+b 후 H, J, K, L)
bind -r H resize-pane -L 5
bind -r J resize-pane -D 5
bind -r K resize-pane -U 5
bind -r L resize-pane -R 5
```

설정 적용:
```bash
tmux source-file ~/.tmux.conf
```

## Part 1: 표준 tmux 기본

### tmux 핵심 구조

```
세션 (Session) - 최상위 컨테이너
  └── 윈도우 (Window) - 브라우저의 탭과 유사
      └── 팬 (Pane) - 분할된 화면
```

### 프리픽스 키

모든 tmux 명령은 **프리픽스 키**로 시작합니다.
- 기본값: `Ctrl + b` (이후 `^b`로 표기)
- 사용법: `^b`를 누른 후 명령 키 입력

예: 수평 분할 → `^b` → `"`

### 1. 세션 관리

```bash
# 새 세션 생성
tmux new -s my-session

# 세션 목록 보기
tmux ls

# 세션에 연결
tmux attach -t my-session

# 세션 분리 (세션은 백그라운드에서 계속 실행)
^b, d

# 세션 종료
tmux kill-session -t my-session

# 모든 세션 종료
tmux kill-server
```

### 2. 창 분할

```bash
# 수평 분할 (위/아래)
^b, "

# 수직 분할 (좌/우)
^b, %

# 팬 간 이동
^b, ↑↓←→

# 팬 닫기
^d 또는 exit
```

### 3. 레이아웃 예제

**2x2 그리드 만들기:**
```bash
^b, "    # 수평 분할
^b, ↑    # 위쪽 팬으로 이동
^b, %    # 수직 분할
^b, ↓    # 아래쪽 팬으로 이동
^b, %    # 수직 분할
```

결과:
```
┌─────────┬─────────┐
│  팬 1   │  팬 2   │
├─────────┼─────────┤
│  팬 3   │  팬 4   │
└─────────┴─────────┘
```

### 4. 사람들 간의 세션 공유

#### 시나리오: 페어 프로그래밍

**개발자 A (호스트):**
```bash
# SSH 서버 또는 공유 머신에서
tmux new -s pair-coding

# 작업 환경 설정
cd ~/project
vim main.py
```

**개발자 B (참여자):**
```bash
# 같은 서버에 접속
ssh user@server

# 세션 목록 확인
tmux ls

# 세션 연결
tmux attach -t pair-coding
```

**이제 두 사람이 같은 화면을 실시간으로 공유! 🎯**

#### 읽기 전용 공유

```bash
# 관찰만 하고 입력은 못하게
tmux attach -t pair-coding -r
```

### 5. 필수 단축키

| 명령 | 설명 |
|------|------|
| `^b, c` | 새 창 생성 |
| `^b, "` | 수평 분할 |
| `^b, %` | 수직 분할 |
| `^b, ↑↓←→` | 팬 이동 |
| `^b, d` | 세션 분리 |
| `^b, [` | 스크롤 모드 (`q`로 종료) |
| `^b, z` | 현재 팬 전체화면 토글 |
| `^b, x` | 현재 팬 종료 (확인) |
| `^b, ?` | 모든 단축키 보기 |

## Part 2: Claude Code Agent Teams

### Agent Teams란?

Claude Code의 **Agent Teams**는 여러 AI 에이전트가 tmux 팬에서 병렬로 작업하는 혁신적인 기능입니다.

**핵심 개념:**
- 🎭 **Team Lead**: 전체 조율하는 메인 에이전트
- 👥 **Teammates**: 특정 작업을 담당하는 팀원 에이전트들
- 💬 **메시징**: 에이전트들이 서로 소통
- 📊 **작업 추적**: 각 팀원의 진행 상황 모니터링

### 1. Agent Teams 활성화

#### 설정 파일 편집

```bash
# Claude Code 설정 파일 경로
~/.claude/settings.json
```

**settings.json에 추가:**

```json
{
  "env": {
    "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1"
  },
  "teammateMode": "tmux"
}
```

**teammateMode 옵션:**
- `"auto"`: tmux 세션 내부면 split pane, 아니면 in-process
- `"tmux"`: 항상 tmux split pane 사용 (권장)
- `"in-process"`: 하나의 터미널에서 모두 실행

#### 또는 환경변수로 설정

```bash
# .zshrc 또는 .bashrc에 추가
export CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1

# 적용
source ~/.zshrc
```

### 2. Claude Code 시작하기

#### iTerm2 + tmux Control Mode (최고의 경험)

```bash
# tmux Control Mode 시작
tmux -CC

# Claude Code 실행
claude
```

**Control Mode의 장점:**
- iTerm2가 tmux 팬을 네이티브 윈도우/탭으로 표시
- 더 깔끔한 UI
- 마우스 클릭으로 팬 전환

#### 표준 tmux 세션

```bash
# tmux 세션 생성
tmux new -s ai-team

# Claude Code 실행
claude
```

### 3. Agent Team 생성하기 ✨

Claude Code 프롬프트에서:

```
Agent Team을 만들어줘.
팀원 1은 백엔드 API 개발
팀원 2는 React 프론트엔드 개발
팀원 3은 테스트 코드 작성
```

**자동으로 일어나는 일:**
1. tmux가 자동으로 3개 팬으로 분할
2. 각 팬에 색상 코드가 지정된 에이전트 생성
3. Team Lead가 각 팀원에게 작업 할당
4. 팀원들이 병렬로 작업 시작!

### 4. 팀원과 상호작용하기

#### 작업 목록 보기
```bash
^b, ^t  # 또는 Ctrl + T
```

#### 특정 팀원 선택 및 메시지
```bash
Shift + ↑/↓  # 팀원 선택
Enter        # 팀원 세션 보기
타이핑       # 직접 메시지 전송
Escape       # 메인으로 돌아가기
```

#### 팀원 인터럽트
```bash
Escape  # 팀원의 현재 작업 중단
```

#### 예제 1: 웹 애플리케이션 개발

### 6. 고급 팀 관리

#### 작업 의존성 설정

```
팀을 만들어줘:
1. Designer: UI mockup 생성 (먼저 완료해야 함)
```


### 8. 서브에이전트 (Subagents) 이해하기

#### 서브에이전트 vs 팀원 (Teammates)

**서브에이전트 (Subagent):**
- 메인 세션 내부에서 실행되는 헬퍼 에이전트
- 작업 완료 후 결과를 메인 에이전트에게 보고
- 독립적인 팬 없음 (같은 컨텍스트 공유)
- 가볍고 빠른 작업에 적합

**팀원 (Teammate):**
- 독립적인 tmux 팬에서 실행
- 자체 컨텍스트 윈도우 보유
- 다른 팀원과 직접 소통 가능
- 복잡하고 긴 작업에 적합

#### 언제 무엇을 사용할까?

```
결정 가이드:

서브에이전트 사용:
✓ 빠른 조사/검색 작업
✓ 파일 검증/확인
✓ 간단한 코드 리뷰
✓ 문서 요약
✓ 단일 작업 완료 후 종료

팀원 사용:
✓ 장기 실행 작업
✓ 여러 파일 편집
✓ 다른 팀원과 협업 필요
✓ 독립적인 개발 모듈
✓ 병렬 작업 필요
```

```
✅ 좋은 서브에이전트:
```

## 비교: 표준 tmux vs Agent Teams vs Subagents

### 세 가지 협업 방식 비교

| 기능 | 표준 tmux | Agent Teams | Subagents |
|------|-----------|-------------|-----------|
| **실행 위치** | 사람이 직접 | 독립 tmux 팬 | 메인 세션 내부 |
| **컨텍스트** | 사람의 두뇌 | 독립 컨텍스트 | 공유 컨텍스트 |
| **병렬 작업** | 불가능 | 가능 | 불가능 |
| **팬 표시** | 사람마다 1개 | 팀원마다 1개 | 없음 |
| **토큰 비용** | $0 | 높음 (팀원 수 × 독립 컨텍스트) | 중간 (메인 컨텍스트 공유) |
| **작업 속도** | 사람 속도 | 빠름 (병렬) | 순차적 (빠름) |
| **적합한 작업** | 페어 프로그래밍 | 대규모 병렬 개발 | 빠른 조사/검증 |

### Subagent vs Teammate 상세 비교

#### Subagent (서브에이전트)

**특징:**
```
✓ 메인 세션 안에서 실행
✓ 메인 에이전트와 컨텍스트 공유
✓ 작업 완료 후 결과 반환
✓ tmux 팬 없음 (백그라운드)
✓ 빠른 시작/종료
```

**장점:**
- ✅ 낮은 토큰 비용
- ✅ 빠른 응답 시간
- ✅ 간단한 작업에 최적
- ✅ 컨텍스트 공유로 효율적

**단점:**
- ❌ 병렬 작업 불가
- ❌ 긴 작업에 부적합
- ❌ 시각적 모니터링 어려움

**사용 예:**
```python
# 서브에이전트 호출 예시
"이 코드를 @code-reviewer에게 빠르게 리뷰받고 결과 알려줘"

# 서브에이전트가 작업하는 동안 다른 작업 불가
# 결과가 메인 에이전트에게 반환됨
```

#### Teammate (팀원)

**특징:**
```
✓ 독립적인 tmux 팬에서 실행
✓ 자체 컨텍스트 윈도우
✓ 다른 팀원과 직접 소통
✓ 실시간 시각적 모니터링
✓ 장시간 작업 가능
```

**장점:**
- ✅ 병렬 작업 가능
- ✅ 복잡한 작업 처리
- ✅ 팀원 간 직접 소통
- ✅ 실시간 진행 상황 확인

**단점:**
- ❌ 높은 토큰 비용
- ❌ 느린 시작 시간
- ❌ 관리 오버헤드

**사용 예:**
```python
# 팀원으로 사용
"@code-reviewer를 독립 팀원으로 스폰해서 
지속적으로 코드 리뷰하게 해줘"

# 팀원이 별도 팬에서 계속 작업
# 다른 팀원들과 병렬로 진행
```

### 최적의 팀 구성 전략

#### 전략 1: 비용 최적화

```
프로젝트 규모: 중소형
예산: 제한적

팀 구성:
1. 서브에이전트 60% - 분석, 검증, 리뷰
2. 팀원 40% - 핵심 개발
3. 사람 - 조율 및 최종 결정

예시:
- @security-auditor (서브) → 빠른 보안 검사
- @code-reviewer (서브) → 코드 리뷰
- Main Developer (팀원) → 핵심 개발
- QA Engineer (팀원) → 통합 테스트
```

#### 전략 2: 속도 최적화

```
프로젝트 규모: 대형
목표: 빠른 개발

팀 구성:
1. 서브에이전트 20% - 빠른 초기 분석만
2. 팀원 80% - 대부분을 병렬 작업
3. 사람 - 고수준 의사결정

예시:
- @security-auditor (서브) → 초기 보안 체크리스트
- Backend Team (팀원 3명) → 병렬 API 개발
- Frontend Team (팀원 2명) → 병렬 UI 개발
- DevOps (팀원) → CI/CD 설정
```

#### 전략 3: 품질 최적화

```
프로젝트 규모: 중형
목표: 높은 품질

팀 구성:
1. 서브에이전트 50% - 지속적 품질 검증
2. 팀원 50% - 개발
3. 사람 - 품질 관리

예시:
- Developer (팀원) → 기능 개발
- @code-reviewer (서브) → 모든 커밋 리뷰
- @test-writer (서브) → 자동 테스트 생성
- @security-auditor (서브) → 보안 스캔
- QA Lead (팀원) → 통합 테스트
```

### 실제 프로젝트 의사결정 트리

```
Q1: 작업이 5분 이내에 끝나는가?
  ├─ Yes → Subagent 사용
  └─ No → Q2로

Q2: 여러 작업을 동시에 해야 하는가?
  ├─ No → Subagent 사용 (순차 처리)
  └─ Yes → Q3으로

Q3: 팀원들끼리 소통해야 하는가?
  ├─ No → 여러 Subagent 순차 실행
  └─ Yes → Teammate 사용

Q4: 예산이 충분한가?
  ├─ Yes → Teammate 최대 활용
  └─ No → 핵심만 Teammate, 나머지 Subagent
```

### 혼합 구성 패턴

#### 패턴 1: 게이트키퍼 (Gatekeeper)

```
Subagent (게이트키퍼) → Teammate (실행자)

예시:
@security-auditor (서브) → 빠른 보안 체크
  ↓ 통과하면
Developer (팀원) → 개발 시작

장점: 비용 효율적인 사전 검증
```

#### 패턴 2: 병렬 개발 + 순차 검증

```
Teammate (병렬) + Subagent (순차)

예시:
Backend (팀원) ┐
Frontend (팀원) ├─ 병렬 개발
Mobile (팀원)  ┘
  ↓ 완료 후
@test-writer (서브) → 순차적으로 테스트 생성

장점: 개발은 빠르게, 검증은 저렴하게
```

#### 패턴 3: 전문가 자문

```
Teammate (개발) ↔ Subagent (자문)

예시:
Developer (팀원) → 개발 중
  ↕ 필요시 문의
@security-auditor (서브) → 즉시 답변

장점: 개발자가 막힐 때만 전문가 활용
```

#### 4. 자동 레이아웃 스크립트

**setup-ai-team.sh:**

```bash
#!/bin/bash

SESSION="ai-dev-team"
PROJECT_DIR="$HOME/projects/my-app"

# 기존 세션 종료
tmux has-session -t $SESSION 2>/dev/null && tmux kill-session -t $SESSION

# 새 세션 생성
tmux new-session -d -s $SESSION -c $PROJECT_DIR

# 팬 분할 (2x2)
tmux split-window -h -t $SESSION
tmux split-window -v -t $SESSION:1.1
tmux split-window -v -t $SESSION:1.2

# 각 팬에서 Claude Code 실행
tmux send-keys -t $SESSION:1.1 "claude" C-m
tmux send-keys -t $SESSION:1.2 "htop" C-m  # 시스템 모니터
tmux send-keys -t $SESSION:1.3 "git status" C-m
tmux send-keys -t $SESSION:1.4 "tail -f logs/development.log" C-m

# 첫 번째 팬 선택
tmux select-pane -t $SESSION:1.1

# 세션 연결
tmux attach -t $SESSION
```

실행:
```bash
chmod +x setup-ai-team.sh
./setup-ai-team.sh
```

**증상:**

```bash
# 3. Claude Code 재시작

^b, :source-file ~/.tmux.conf
```

**해결:**

```bash
# 수동으로 팬 크기 조절

^b, :resize-pane -U 10  # 위로 10칸

# 또는 레이아웃 변경
```

#### 문제 3: 팀원 세션이 남아있음

**증상:**
```bash
tmux ls
# team-abc-backend: 1 windows (still running)
```

**해결:**
```bash
# 특정 세션 종료
tmux kill-session -t team-abc-backend

# 패턴으로 세션 종료
tmux list-sessions | grep "team-" | cut -d: -f1 | xargs -I {} tmux kill-session -t {}
```

#### 문제 4: 색상이 제대로 표시되지 않음

**해결:**
```bash
# .tmux.conf에 추가

# 설정 적용
tmux source-file ~/.tmux.conf
```

## 마치며

이 튜토리얼로 세 가지 강력한 협업 방식을 배웠습니다:

**🤝 표준 tmux:**
- 원격 팀원과 실시간 협업
- 세션 공유를 통한 페어 프로그래밍
- 안정적이고 검증된 워크플로우

**👥 서브에이전트 (Subagents):**
- 빠르고 저렴한 AI 작업 자동화
- 코드 리뷰, 테스트 작성, 문서화
- 순차적이지만 효율적인 실행

**🤖 Claude Code Agent Teams:**
- AI 에이전트들의 병렬 작업
- 자동화된 개발 프로세스
- 대규모 프로젝트 빠른 처리

세 방식을 **조합**하면 최고의 생산성을 얻을 수 있습니다:
- 사람은 고수준 의사결정
- 서브에이전트가 빠른 검증
- 팀원들이 병렬로 구현
- tmux로 모든 작업 시각화

### 핵심 요약

```
1. tmux = 터미널 멀티플렉서 (기반)
2. 표준 tmux = 사람들 간 협업
3. Claude Code = AI 코딩 도구
4. Subagents = 빠른 AI 작업 (순차)
5. Agent Teams = AI 팀 협업 (병렬)
6. tmux + Subagents + Teams = 최강 조합
```

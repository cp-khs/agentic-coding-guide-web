# MoAI Skills 참조 가이드

> **목적**: Claude Code에서 사용 가능한 MoAI 스킬 전체 목록과 사용 시점 가이드
> **사용법**: `/skill-name` 또는 `Skill("skill-name")` 도구로 호출
> **개념 이해** (아키텍처·워크플로우·에이전트·LLM 교체·타 프레임워크 비교): [MoAI ADK 가이드](/tools/moai/adk-guide)

---

## 스킬이란?

MoAI 스킬(Skill)은 `.claude/skills/` 디렉토리에 저장된 마크다운 파일로,
호출 시 해당 도메인의 전문 지식과 패턴이 Claude의 컨텍스트에 **주입**됩니다.

```
# 호출 방법
/moai plan           → moai 스킬이 로드되어 Intent Router 활성화
/explain-code        → explain-code 스킬이 로드되어 시각적 코드 설명 모드 활성화

# Skill 도구로 직접 호출
Skill("moai-lang-python")     → Python 전문가 컨텍스트 주입
Skill("moai-domain-backend")  → 백엔드 아키텍처 전문가 컨텍스트 주입
```

### 1. 코어 오케스트레이터

핵심 MoAI 시스템을 구성하는 스킬들입니다.

| 스킬 | 사용 시점 |
|------|-----------|
| **moai** | 모든 개발 작업의 진입점. plan/run/sync/fix/loop/project/feedback 서브커맨드 라우팅 |
| **moai-foundation-core** | TRUST 5 품질 프레임워크, SPEC-First DDD 방법론, 에이전트 위임 패턴 참조 시 |

| **moai-foundation-quality** | 코드 리뷰, 품질 게이트 검증, lint 설정, TRUST 5 컴플라이언스 시 |
| **moai-foundation-thinking** | 아이디어 생성, 제안 평가, 가정 검토, 솔루션 탐색 등 창의적 문제 해결 시 |
| **moai-foundation-philosopher** | 아키텍처 결정, 기술 선택 트레이드오프, 근본 원인 분석, 인지 편향 탐지 시 |

## 스킬 선택 가이드

```
작업 유형에 따른 스킬 선택:

새 기능 기획         → moai plan  (또는 moai-workflow-spec)
코드 구현            → moai run   (또는 moai-workflow-ddd / moai-workflow-tdd)
버그 수정            → moai fix
반복 오류 수정       → moai loop  (또는 moai-workflow-loop)
문서 업데이트        → moai sync  (또는 moai-docs-generation)

언어별 심화 작업:
  Python FastAPI     → moai-lang-python
  React/Next.js      → moai-lang-typescript + moai-domain-frontend
  PostgreSQL 최적화  → moai-domain-database

아키텍처 결정        → moai-foundation-philosopher
복잡한 문제 분석     → moai-workflow-thinking (--ultrathink 플래그)
코드 품질 검증       → moai-foundation-quality

팀 협업 시작         → /create-team [팀이름]
```

## /moai 서브커맨드 요약

| 서브커맨드 | 별칭 | 목적 |
|-----------|------|------|
| `/moai plan` | spec | SPEC 문서 작성 (EARS 형식) |
| `/moai run` | impl | SPEC 기반 DDD 구현 |
| `/moai sync` | docs, pr | 문서 동기화 + PR 생성 |
| `/moai fix` | - | 오류 자동 수정 (1회) |
| `/moai loop` | - | 반복 자동 수정 (완료까지) |
| `/moai project` | init | 프로젝트 문서 생성 |
| `/moai feedback` | fb, bug, issue | GitHub 이슈 생성 |
| `/moai [자연어]` | - | 전체 자율 파이프라인 (plan→run→sync) |

## 실행 모드 플래그

모든 `/moai` 커맨드에 적용 가능한 플래그:

| 플래그 | 효과 |
|--------|------|
| `--team` | Agent Teams 모드 강제 (병렬 실행) |
| `--solo` | 서브에이전트 모드 강제 (단일 에이전트) |
| `--ultrathink` | Sequential Thinking MCP 활성화 (깊은 분석) |
| `--resume SPEC-XXX` | 중단된 워크플로우 재개 |
| `--worktree` | 격리된 git worktree에서 실행 |
| `--branch` | 기능 브랜치 생성 |
| `--pr` | PR 자동 생성 |
| `--loop` | 반복 수정 활성화 |
| `--max N` | 최대 반복 횟수 설정 |

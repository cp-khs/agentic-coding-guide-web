## 1. 개요: 세 가지 사용 모드

Claude Code는 크게 세 가지 방식으로 사용할 수 있다.

```
┌─────────────────────────────────────────────────────────────┐
│               Claude Code 사용 모드                          │
│                                                             │
│  ① 터미널 단독 모드          ② VS Code 확장 모드             │
│  ┌──────────────────┐       ┌──────────────────────┐        │
│  │ $ claude         │       │ ┌─────────┬────────┐ │        │
│  │ > 지시사항 입력   │       │ │ 에디터  │ Claude │ │        │
│  │                  │       │ │         │  패널  │ │        │
│  │ 텍스트 기반 출력  │       │ │ 코드    │  대화  │ │        │
│  │ 키보드 중심 조작  │       │ │ 편집    │  UI   │ │        │
│  └──────────────────┘       │ └─────────┴────────┘ │        │
│                             └──────────────────────┘        │
│                                                             │
│  ③ IDE 내장 터미널 모드                                      │
│  ┌──────────────────────────────────────┐                   │
│  │ ┌─────────────────────────────────┐  │                   │
│  │ │         에디터 영역              │  │                   │
│  │ ├─────────────────────────────────┤  │                   │
│  │ │ $ claude                        │  │                   │
│  │ │ 터미널 CLI를 IDE 안에서 실행     │  │                   │
│  │ └─────────────────────────────────┘  │                   │
│  └──────────────────────────────────────┘                   │
└─────────────────────────────────────────────────────────────┘
```

| 모드 | 인터페이스 | 주요 특징 |
|------|-----------|----------|
| ① 터미널 단독 | iTerm2, Terminal.app 등 | 가볍고 빠름, 키보드 중심 |
| ② VS Code 확장 | VS Code 사이드 패널 | 시각적 diff, 그래픽 UI |
| ③ IDE 내장 터미널 | VS Code/Cursor 내장 터미널 | ①과 ②의 절충 |

## 3. 터미널 단독 모드

### 3.1 개요

외부 터미널(iTerm2, Terminal.app, Windows Terminal 등)에서 `claude`를 실행하는 방식이다. IDE를 전혀 사용하지 않거나, 별도 모니터에 IDE를 띄워두고 터미널에서만 Claude와 상호작용한다.

### 3.2 시작 방법

```bash
# 프로젝트 디렉토리로 이동 후 실행
cd ~/projects/my-app
claude
```

### 3.3 장점

**집중력과 몰입**
- 시각적 방해 요소가 없는 미니멀한 환경
- "Deep Work" 모드에 최적화
- 터미널 하나에 모든 것이 집중됨

**키보드 중심의 효율성**
- `Tab`: 생각 모드(Extended Thinking) 전환
- `Shift+Tab` (2회): Plan Mode 전환
- `Double-ESC`: 이전 메시지로 되감기 (Context Rewind)
- `Ctrl+R`: 대화 히스토리 검색
- `/clear`, `/status`, `/usage` 등 빠른 명령어

**경량 리소스**
- IDE를 실행하지 않으므로 메모리와 CPU 사용량이 최소
- 저사양 환경이나 원격 서버에서도 원활히 동작
- SSH로 원격 서버에 접속해서 그대로 사용 가능

**고급 기능 완전 지원**
- 모든 슬래시 명령어 사용 가능 (`/`, `!` 등)
- MCP 서버 설정 직접 가능
- `--dangerously-skip-permissions` 플래그로 자동화 파이프라인 구성
- `!` 접두사로 쉘 명령 직접 실행
- Tab 자동완성 지원

**에이전트 팀과의 궁합**
- 에이전트 팀 기능은 터미널 환경에서 설계되었음
- tmux 또는 iTerm2 분할 패널로 여러 에이전트 세션을 동시에 모니터링 가능
- 다중 터미널 탭/패널로 작업 분리가 자연스러움

### 3.4 단점

**시각적 피드백 부족**
- 코드 변경사항을 인라인 diff로 볼 수 없음
- 파일 변경 전후를 확인하려면 별도의 `git diff` 실행 필요
- 대규모 코드 변경 시 변경 내용 파악이 어려움

**파일 탐색의 불편함**
- 프로젝트 구조를 GUI로 한눈에 볼 수 없음
- 파일 간 이동이 CLI 명령에 의존

**초보자 진입 장벽**
- 터미널 사용 경험이 없으면 학습 곡선이 가파름
- 명령어와 단축키를 외워야 함
- 실수 시 되돌리기가 직관적이지 않음

### 3.5 최적 사용법

```bash
# 1. tmux로 작업 환경 구성 (터미널 단독 모드의 핵심)
tmux new-session -s dev

# 왼쪽: Claude Code 실행
claude

# 오른쪽 (Ctrl+B %): 파일 변경 모니터링
watch -n 1 git status

# 또는 오른쪽에서 수시로 diff 확인
git diff
```

**추천 터미널 세팅**:
- 터미널 에뮬레이터: iTerm2 (macOS) 또는 Alacritty
- 세션 관리: tmux
- 폰트: JetBrains Mono Nerd Font (아이콘 지원)
- 최소 80x40 터미널 크기 권장

## 4. VS Code 확장 모드

### 4.1 개요

VS Code Marketplace에서 "Claude Code" 확장을 설치하면 IDE 내에 전용 그래픽 패널이 추가된다. 코드 에디터와 나란히 놓고 시각적으로 상호작용하는 방식이다.

### 4.2 시작 방법

1. VS Code에서 `Cmd+Shift+X` → "Claude Code" 검색 → 설치
2. 에디터 우상단의 Spark(✱) 아이콘 클릭
3. Anthropic 계정으로 로그인

Cursor에서도 동일하게 설치 가능하다:
- Cursor에서 `cursor:extension/anthropic.claude-code`로 설치

### 4.3 장점

**시각적 코드 리뷰**
- Claude가 파일을 수정하면 인라인 diff가 즉시 표시됨
- 초록색(추가), 빨간색(삭제), 파란색(수정)으로 색상 구분
- 변경사항을 라인 단위로 Accept/Reject 가능
- VS Code의 Source Control 패널에서 side-by-side 비교 가능

**Plan Mode의 시각적 편집**
- Claude의 계획을 GUI에서 직접 편집한 후 실행 가능
- Normal / Plan / Auto-accept 세 가지 권한 모드를 UI에서 전환
- 프롬프트 박스 하단의 모드 표시기로 현재 상태 확인

**@-멘션을 통한 정밀한 컨텍스트 제공**
- `@filename`으로 특정 파일 참조
- `@filename#5-10`으로 특정 라인 범위 참조
- `Option+K` (Mac) / `Alt+K`로 현재 선택 영역을 자동으로 @-멘션 삽입
- 에디터에서 텍스트를 선택하면 Claude가 자동으로 인식

**다중 대화 및 세션 관리**
- 여러 대화를 별도 탭이나 창에서 동시에 실행 가능
- 과거 대화 히스토리를 검색하고 재개 가능
- 탭의 색상 점으로 상태 확인 (파란색: 권한 요청 대기, 주황색: 작업 완료)

**브라우저 자동화**
- `@browser` 키워드로 Chrome 브라우저 작업 자동화
- 웹 앱 테스트, 콘솔 로그 확인 등 가능

**체크포인트(Rewind) 지원**
- 메시지 위에 마우스를 올리면 되감기 버튼 표시
- "Fork conversation from here": 특정 시점부터 새로운 대화 분기
- "Rewind code to here": 코드를 특정 시점으로 되돌림
- "Fork conversation and rewind code": 분기 + 코드 되돌림

### 4.4 단점

**리소스 사용량**
- VS Code 자체가 메모리를 상당량 사용
- 대규모 프로젝트에서는 체감 가능한 성능 저하 가능

**일부 CLI 기능 미지원**
- `!` 접두사로 쉘 명령 직접 실행 불가
- Tab 자동완성 미지원
- MCP 서버 설정은 CLI에서만 가능 (설정 후 확장에서 사용은 가능)
- 일부 슬래시 명령어가 확장에서 지원되지 않음

**에이전트 팀 모니터링의 한계**
- 에이전트 팀의 여러 세션을 동시에 시각적으로 모니터링하기 어려움
- 터미널 분할 기반의 팀 운영은 IDE 내에서 덜 직관적

### 4.5 최적 사용법

```
추천 레이아웃:

┌─────────────────────────────────┬─────────────────┐
│                                 │                 │
│         에디터 영역              │   Claude Code   │
│     (파일 편집 / diff 확인)      │     사이드바     │
│                                 │                 │
│                                 │   대화 + 계획    │
│                                 │   + diff 리뷰   │
│                                 │                 │
├─────────────────────────────────┴─────────────────┤
│                  터미널 패널                        │
│         (테스트 실행, git 명령 등)                   │
└───────────────────────────────────────────────────┘
```

- Claude 패널을 오른쪽 사이드바에 배치 (기본 설정)
- 에디터에서 코드를 선택한 뒤 `Option+K`로 참조 삽입
- Auto-accept 모드는 신뢰가 쌓인 후에 활성화
- 하단 터미널은 테스트 실행과 git 작업에 활용

## 5. IDE 내장 터미널 모드

### 5.1 개요

VS Code나 Cursor의 내장 터미널에서 `claude`를 실행하는 방식이다. 터미널 단독 모드의 기능을 IDE 안에서 사용하면서도 에디터와 파일 탐색기의 이점을 함께 누릴 수 있다.

### 5.2 시작 방법

**방법 1: 내장 터미널에서 직접 실행**
```bash
# VS Code/Cursor에서 Ctrl+` (터미널 열기)
claude
```

**방법 2: VS Code 확장에서 터미널 모드로 전환**
- VS Code 설정 → Extensions → Claude Code → "Use Terminal" 체크
- 또는 Command Palette에서 "Claude Code: Open in Terminal" 실행

**방법 3: 외부 터미널에서 IDE 연결**
```bash
# 외부 터미널에서 Claude를 실행한 뒤 IDE에 연결
claude
/ide
```

### 5.3 장점

**터미널 CLI의 전체 기능 + IDE의 시각적 편의**
- 모든 CLI 명령어와 단축키를 그대로 사용 가능
- 파일 변경 시 IDE의 에디터에서 즉시 확인
- IDE의 Source Control(git) 패널로 diff 확인
- 파일 탐색기로 프로젝트 구조를 시각적으로 확인

**자동 IDE 통합**
- 내장 터미널에서 `claude`를 실행하면 자동으로 IDE와 연동
- diff 뷰잉과 진단 정보(linter 에러 등)가 공유됨
- `@terminal:name`으로 터미널 출력을 프롬프트에 참조 가능

**유연한 레이아웃**
- IDE 내에서 여러 터미널 탭을 열어 복수의 Claude 세션 운영 가능
- 에디터, 터미널, 파일 탐색기를 자유롭게 배치

### 5.4 단점

**확장 모드의 전용 기능 미사용**
- 인라인 diff의 Accept/Reject 버튼 없음
- Plan Mode의 GUI 편집 기능 없음
- @-멘션 자동완성이 덜 편리함
- 체크포인트 GUI 버튼 없음 (CLI 명령으로는 가능)

**공간 제약**
- IDE 하단 터미널 영역은 에디터보다 좁음
- 긴 출력이 있을 때 스크롤이 불편할 수 있음

### 5.5 최적 사용법

```
추천 레이아웃:

┌───────────────────────────────────────────────────┐
│                                                   │
│              에디터 영역 (넓게)                     │
│         코드 확인, diff 리뷰, 파일 탐색             │
│                                                   │
├───────────────────────────────────────────────────┤
│  $ claude                                         │
│  > 인증 API를 구현해줘...                           │
│                                                   │
│  터미널 영역 (적절한 높이 확보)                      │
└───────────────────────────────────────────────────┘
```

- 터미널 영역의 높이를 충분히 확보 (화면의 40% 정도)
- 에디터에서 변경된 파일을 수시로 확인
- VS Code의 Source Control 패널을 활용해서 diff 확인

## 8. 하이브리드 워크플로우

실무에서는 하나의 모드만 고집하기보다, **작업 단계에 따라 모드를 전환**하는 것이 가장 효과적이다.

### 8.1 권장 하이브리드 패턴

```
┌──────────────────────────────────────────────────────────┐
│              하이브리드 워크플로우                           │
│                                                          │
│  [설계 단계]                                              │
│     VS Code 확장 모드 (Plan Mode)                         │
│     → 시각적으로 계획 검토 및 편집                          │
│                    │                                      │
│  [구현 단계]                                              │
│     터미널 단독 또는 IDE 내장 터미널                        │
│     → 집중해서 코드 생성, 에이전트 자율 실행                 │
│                    │                                      │
│  [리뷰 단계]                                              │
│     VS Code 확장 모드                                     │
│     → 인라인 diff로 변경사항 리뷰                          │
│                    │                                      │
│  [멀티 에이전트 단계]                                      │
│     터미널 단독 (tmux 분할)                                │
│     → 에이전트 팀 모니터링                                 │
│                    │                                      │
│  [검증 단계]                                              │
│     IDE 내장 터미널                                        │
│     → 테스트 실행 + 결과 확인을 IDE에서 함께                 │
└──────────────────────────────────────────────────────────┘
```

### 8.2 듀얼 모니터 활용

듀얼 모니터 환경이라면 가장 강력한 조합은 다음과 같다:

```
모니터 1 (IDE)                    모니터 2 (터미널)
┌─────────────────────┐          ┌─────────────────────┐
│                     │          │  $ claude            │
│  VS Code / Cursor   │          │  > 지시 입력         │
│                     │          │                     │
│  - 파일 탐색기       │          │  Claude 메인 세션    │
│  - 에디터 (diff)     │          │                     │
│  - Source Control    │          ├─────────────────────┤
│  - 진단 패널         │          │                     │
│                     │          │  추가 터미널/테스트   │
│                     │          │                     │
└─────────────────────┘          └─────────────────────┘
```

- 모니터 1: IDE로 코드 변경 확인, 파일 탐색, diff 리뷰
- 모니터 2: 터미널에서 Claude와 상호작용, 테스트 실행

## 9. Cursor와 함께 사용하기

### 9.1 Cursor에서 Claude Code 사용하기

Cursor는 자체 AI 기능(Cursor AI)이 내장되어 있지만, Claude Code도 함께 사용할 수 있다.

**방법 1: Claude Code 확장 설치**
```
# Cursor에서 확장 설치
cursor:extension/anthropic.claude-code
```

**방법 2: 내장 터미널에서 실행**
```bash
# Cursor의 내장 터미널 열기 (Ctrl+`)
claude
```

**방법 3: 외부 터미널에서 Cursor에 연결**
```bash
# 외부 터미널에서 Claude 실행 후 Cursor에 연결
claude
/ide
```

### 9.2 Cursor AI vs Claude Code: 역할 분담

Cursor의 자체 AI와 Claude Code를 동시에 사용할 때는 역할을 분담하는 것이 효과적이다.

| 작업 | Cursor AI | Claude Code |
|------|:---------:|:-----------:|
| 인라인 코드 수정 (한두 줄) | ✅ | |
| 자동완성 | ✅ | |
| 대규모 리팩토링 | | ✅ |
| 멀티 파일 생성 | | ✅ |
| 쉘 명령 실행 | | ✅ |
| 서브에이전트 위임 | | ✅ |
| 에이전트 팀 구성 | | ✅ |
| 코드 질문/설명 | ✅ | ✅ |
| 테스트 작성 + 실행 + 수정 | | ✅ |
| git 작업 자동화 | | ✅ |

### 9.3 Cursor에서의 추천 워크플로우

```
일상적 코딩                      복잡한 작업
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Cursor AI 자동완성              Claude Code 터미널
  ↓ 한두 줄 수정                  ↓ 대규모 변경
Cursor AI 인라인 편집            Claude Code 서브에이전트
  ↓ 빠른 질문                    ↓ 멀티 에이전트 협업
Cursor AI 채팅                  Claude Code 에이전트 팀
```

## 10. 모드 전환 및 세션 공유

### 10.1 세션 공유

VS Code 확장과 CLI는 **동일한 대화 히스토리를 공유**한다. 확장에서 시작한 대화를 CLI에서 이어갈 수 있고, 그 반대도 가능하다.

```bash
# VS Code 확장에서 시작한 대화를 CLI에서 이어가기
claude --resume
# → 대화 목록에서 선택
```

### 10.2 모드 전환

**확장 → 터미널**
- VS Code 설정에서 "Use Terminal" 체크
- 또는 Command Palette에서 "Claude Code: Open in Terminal"

**터미널 → 확장**
- VS Code 설정에서 "Use Terminal" 해제
- Spark 아이콘 클릭

**외부 터미널 → IDE 연결**
```bash
# Claude Code 실행 중에 IDE 연결
/ide
```

### 10.3 주의 사항

- 한 프로젝트에서 여러 Claude 세션을 실행할 때 **같은 파일을 동시에 수정하면 충돌** 발생 가능
- 에이전트 팀의 경우 파일 잠금이 없으므로 "마지막 쓰기가 이김"
- 세션 전환 시 컨텍스트 윈도우 사용량을 확인할 것 (`/usage`)

## 11. 정리 및 추천

### 개발자 프로필별 추천

| 프로필 | 추천 모드 | 이유 |
|--------|----------|------|
| 터미널 숙련자 (vim/tmux 사용자) | 터미널 단독 | 기존 워크플로우와 자연스러운 통합 |
| VS Code 주 사용자 | VS Code 확장 | 익숙한 환경에서 시각적 피드백 활용 |
| Cursor 사용자 | IDE 내장 터미널 + Cursor AI | 두 AI의 장점을 역할 분담으로 활용 |
| 에이전틱 코딩 초보자 | VS Code 확장 | 직관적 UI로 빠른 학습 |
| 원격 서버 개발자 | 터미널 단독 | SSH만으로 사용 가능 |
| 멀티 에이전트 활용자 | 터미널 단독 (tmux) | 에이전트 팀에 최적화 |

### 핵심 원칙

1. **하나의 모드에 고정하지 마라**: 작업 성격에 따라 모드를 전환하는 것이 가장 효과적이다.
2. **시각적 리뷰가 필요하면 확장, 집중이 필요하면 터미널**: 이 기본 원칙만 기억하면 된다.
3. **에이전트 팀은 터미널에서**: 멀티 에이전트 워크플로우는 터미널 환경에서 설계되었다.
4. **세션 공유를 활용하라**: 확장과 CLI의 대화 히스토리는 공유되므로 자유롭게 전환 가능하다.
5. **초보자는 확장에서 시작하고 점차 터미널로**: 시각적 피드백으로 학습한 뒤 터미널의 효율성을 탐색하라.

## 1. 메모리 시스템 개요

Claude Code는 세션이 끝나도 컨텍스트를 유지할 수 있는 두 가지 메모리 시스템을 제공한다.

```
┌─────────────────────────────────────────────────────────────┐
│               Claude Code 메모리 시스템                       │
│                                                             │
│  ① CLAUDE.md / Rules     ② Auto Memory                     │
│  ┌──────────────────┐    ┌──────────────────────┐           │
│  │ 사람이 직접 작성  │    │ Claude가 자동으로     │           │
│  │ 프로젝트 규칙     │    │ 학습 내용 저장        │           │
│  │ 코딩 컨벤션       │    │                      │           │
│  │ 개인 설정         │    │ 프로젝트 패턴         │           │
│  │ 팀 표준           │    │ 디버깅 노트           │           │
│  └──────────────────┘    └──────────────────────┘           │
│                                                             │
│  세션 시작 시 자동 로드 → Claude의 시스템 프롬프트에 포함      │
└─────────────────────────────────────────────────────────────┘
```

## 2. 파일 종류와 적용 범위

| 파일 위치 | 적용 범위 | 공유 여부 |
|---|---|---|
| `/Library/Application Support/ClaudeCode/CLAUDE.md` | 조직 전체 (IT 정책) | 조직 내 모든 사용자 |
| `~/.claude/CLAUDE.md` | 내 모든 프로젝트 | 나만 |
| `~/.claude/rules/*.md` | 내 모든 프로젝트 (조건부 가능) | 나만 |
| `./CLAUDE.md` 또는 `./.claude/CLAUDE.md` | 현재 프로젝트 | 팀 (git 공유) |
| `./.claude/rules/*.md` | 현재 프로젝트 (조건부 가능) | 팀 (git 공유) |
| `./CLAUDE.local.md` | 현재 프로젝트 | 나만 (자동 gitignore) |
| `~/.claude/projects/<project>/memory/MEMORY.md` | 현재 프로젝트 | 나만 (Auto Memory) |

### 어떤 파일을 써야 할까?

```
새 규칙을 추가하려 한다
         │
         ▼
모든 프로젝트에 적용? ─── YES ──→ ~/.claude/rules/<topic>.md
         │
         NO
         │
         ▼
팀과 공유? ─── YES ──→ ./CLAUDE.md 또는 ./.claude/rules/<topic>.md
         │
         NO
         │
         ▼
나만 사용 → ./CLAUDE.local.md
```

## 3. 우선순위와 로드 순서

파일들은 **모두 누적**되어 시스템 프롬프트에 포함된다. 덮어쓰지 않는다.
충돌하는 지침이 있을 때는 **더 구체적인(나중에 로드된) 것이 우선**된다.

```
낮은 우선순위
     │
     ▼  1. 조직 정책  (/Library/.../CLAUDE.md)
     │
     ▼  2. 개인 전역  (~/.claude/CLAUDE.md)
     │
     ▼  3. 개인 rules (~/.claude/rules/*.md)
     │
     ▼  4. 프로젝트   (./CLAUDE.md, ./.claude/CLAUDE.md)
     │
     ▼  5. 프로젝트 rules (./.claude/rules/*.md)
     │
     ▼  6. 로컬 개인  (./CLAUDE.local.md)
     │
높은 우선순위
```

**실전 시나리오**: `~/.claude/rules/language.md`에 "한국어 응답" 규칙이 있어도,
프로젝트의 `CLAUDE.md`에 "영어로 응답"이라고 적혀 있으면 프로젝트 설정이 이긴다.

## 4. CLAUDE.md 파일 활용

### 프로젝트 CLAUDE.md 기본 구조

```markdown
# 프로젝트명

## 프로젝트 개요
- 기술 스택, 아키텍처, 주요 의존성

## 코딩 컨벤션
- 네이밍 규칙, 들여쓰기, 주석 스타일

## 자주 사용하는 명령어
- 빌드, 테스트, 실행 명령어 (매번 검색하지 않도록)

## 아키텍처 결정 사항
- 중요한 설계 결정과 그 이유
```

### @import 문법으로 파일 분리

CLAUDE.md가 커지면 외부 파일로 분리할 수 있다.

```markdown
# 프로젝트 설정

프로젝트 개요는 @README.md 참고.
사용 가능한 명령어는 @package.json 참고.

## 추가 규칙
- Git 워크플로우: @docs/git-workflow.md
```

### CLAUDE.local.md — 나만의 로컬 설정

팀과 공유하지 않을 개인 설정은 `CLAUDE.local.md`에 작성한다.
자동으로 `.gitignore`에 추가되어 커밋되지 않는다.

```markdown
# 로컬 개인 설정

## 개발 환경
- 로컬 DB: postgresql://localhost:5432/mydb_dev
- 테스트 API 키: 내 개인 샌드박스 키 사용

## 개인 선호
- 주석은 한국어로 작성
```

## 5. Rules 시스템 활용

`.claude/rules/` 디렉토리의 파일들은 자동으로 탐색된다.
파일명은 중요하지 않고, **`paths` frontmatter 유무**가 적용 범위를 결정한다.

### 항상 적용되는 규칙 (frontmatter 없음)

```markdown
# language.md

## 응답 언어
- 모든 응답을 한국어로 작성
- 기술 용어와 코드 식별자는 원래 형태 유지
```

### 특정 파일 타입에만 적용되는 규칙

```markdown
---
paths:
  - "**/*.py"
  - "**/pyproject.toml"
---

# Python 개발 규칙

- Python 3.12+ 문법 사용
- 타입 힌트 필수
- 포매터: ruff
- 테스트: pytest, 커버리지 85% 이상
```

### 디렉토리 구조 예시

```
~/.claude/rules/          ← 개인 전역 규칙
├── language.md           ← 항상 적용 (응답 언어)
├── code-style.md         ← 항상 적용 (일반 코딩 스타일)
└── python.md             ← .py 파일 작업 시만 적용

.claude/rules/            ← 프로젝트 규칙 (팀 공유)
├── architecture.md       ← 항상 적용 (아키텍처 결정)
├── testing.md            ← 항상 적용 (테스트 기준)
└── api-conventions.md    ← paths로 API 파일에만 적용
```

### 지원하는 Glob 패턴

| 패턴 | 매칭 대상 |
|---|---|
| `**/*.py` | 모든 Python 파일 |
| `src/**/*` | src/ 하위 모든 파일 |
| `**/*.{ts,tsx}` | TypeScript/TSX 파일 |
| `{src,lib}/**/*.ts` | src 또는 lib 하위 TS 파일 |
| `*.md` | 루트 디렉토리 마크다운 |

## 6. Auto Memory 활용

Claude가 세션 중에 스스로 학습 내용을 저장하는 시스템이다.
위치: `~/.claude/projects/<project-path>/memory/`

```
~/.claude/projects/<project>/memory/
├── MEMORY.md          ← 인덱스 (세션 시작 시 자동 로드, 앞 200줄만)
├── debugging.md       ← 디버깅 패턴 상세 노트
├── api-conventions.md ← API 설계 결정사항
└── patterns.md        ← 프로젝트 코드 패턴
```

### 활성화 방법

Auto Memory는 단계적으로 배포 중이다. 강제 활성화하려면:

```bash
export CLAUDE_CODE_DISABLE_AUTO_MEMORY=0   # 활성화
export CLAUDE_CODE_DISABLE_AUTO_MEMORY=1   # 비활성화
```

### 직접 저장 요청

Claude에게 직접 기억하도록 요청할 수 있다.

```
"pnpm을 사용한다고 기억해줘"
"로컬 Redis 인스턴스가 필요하다는 걸 저장해줘"
"이 버그의 해결 방법을 메모리에 남겨줘"
```

### MEMORY.md 작성 가이드

- 앞 200줄만 자동 로드되므로 **간결하게 유지**
- 상세 내용은 별도 파일(`debugging.md` 등)로 분리
- 날짜 없이 내용 중심으로 작성 (날짜는 빠르게 낡음)

## 7. 현재 로드된 메모리 확인

세션 중 어떤 파일이 로드됐는지 확인하려면:

```
/memory
```

세션 ID와 태스크 저장 위치 확인:

```bash
ls -lt ~/.claude/tasks/ | head -3
# 가장 최근 수정된 디렉토리 이름 = 현재 세션 ID
```

세션 재개:

```
/resume <세션-ID>
```

## 8. 실전 설정 예시

### 개인 전역 설정 (모든 프로젝트 적용)

```
~/.claude/
├── CLAUDE.md              ← 기타 전역 지침 (선택)
└── rules/
    ├── language.md        ← 응답 언어 설정
    ├── git.md             ← 커밋 메시지 형식
    └── preferences.md     ← 패키지 매니저, 도구 선호
```

**`~/.claude/rules/language.md`**
```markdown
## 응답 언어 (HARD)
- 모든 응답(사고 과정 포함)을 한국어로 작성
- 기술 용어와 코드 식별자는 원래 형태 유지
```

**`~/.claude/rules/git.md`**
```markdown
## Git 커밋 규칙
- Conventional Commits 형식 사용 (feat:, fix:, docs: 등)
- 커밋 메시지는 한국어로 작성
- 커밋 전 자동 push 금지, 반드시 확인 요청
```

### 프로젝트별 설정 (팀 공유)

```
.claude/
├── CLAUDE.md              ← 프로젝트 핵심 설정
└── rules/
    ├── architecture.md    ← 아키텍처 결정
    ├── testing.md         ← 테스트 기준
    └── backend.md         ← 백엔드 파일에만 적용
```

## 9. 안티패턴

### ❌ CLAUDE.md에 모든 것을 때려넣기

```markdown
# CLAUDE.md (나쁜 예 - 수백 줄)
## 언어 설정
## Python 규칙
## TypeScript 규칙
## Git 규칙
## 테스트 규칙
## 아키텍처
...
```

**→ 해결**: Rules 시스템으로 파일을 분리하고 `paths` frontmatter로 필요할 때만 로드

---

### ❌ 같은 내용을 여러 파일에 중복 작성

한국어 지침을 `~/.claude/CLAUDE.md`, `~/.claude/rules/language.md`, `MEMORY.md`에 모두 작성하면 토큰 낭비이고 관리가 어렵다.

**→ 해결**: 한 곳에만 작성. 역할별로 파일을 구분한다.

---

### ❌ Auto Memory와 CLAUDE.md 혼용

Auto Memory는 Claude가 자동으로 관리하는 공간이다.
사람이 직접 규칙을 적으려면 CLAUDE.md나 Rules 파일을 사용한다.

---

### ❌ 개인 정보를 프로젝트 CLAUDE.md에 작성

DB 비밀번호, API 키, 개인 환경 경로 등은 `CLAUDE.local.md`에 작성해야 한다.
`CLAUDE.md`는 git에 커밋되어 팀 전체에 공유된다.

---

## 정리

| 목적 | 사용할 파일 |
|---|---|
| 모든 프로젝트에 적용할 개인 규칙 | `~/.claude/rules/<topic>.md` |
| 특정 파일 타입에만 적용할 규칙 | `~/.claude/rules/<topic>.md` (paths frontmatter) |
| 팀과 공유할 프로젝트 규칙 | `./.claude/rules/<topic>.md` |
| 나만 사용할 프로젝트 설정 | `./CLAUDE.local.md` |
| Claude의 자동 학습 노트 | Auto Memory (자동 관리) |

---

> **참고**: `/memory` 명령으로 현재 세션에 로드된 메모리 파일 전체 목록을 확인할 수 있다.
> 공식 문서: [https://code.claude.com/docs/en/memory](https://code.claude.com/docs/en/memory)

# TaskFlow 튜토리얼 실행 가이드

Claude Code를 활용하여 TaskFlow 프로젝트를 단계별로 직접 실행하며 학습합니다.

## 🎯 개요

TaskFlow는 팀 태스크 관리 웹 애플리케이션으로, 다음 기술 스택을 사용합니다:
- **백엔드**: Python 3.12, FastAPI, SQLAlchemy, PostgreSQL
- **프론트엔드**: Next.js 15, TypeScript, Tailwind CSS
- **인프라**: Docker, Docker Compose
- **기능**: 인증, 프로젝트 관리, 칸반 보드, 대시보드

---

## 🎓 튜토리얼 Phase

전체 튜토리얼은 5개 Phase로 구성됩니다:

| Phase | 내용 | 예상 시간 | 주요 학습 내용 |
|-------|------|-----------|----------------|
| **1. 프로젝트 구성** | CLAUDE.md, 디렉토리 구조, 서브에이전트 | 30분 | 프로젝트 초기 설정, 에이전트 정의 |
| **2. 백엔드 구현** | FastAPI, SQLAlchemy, JWT 인증, API | 60분 | 백엔드 아키텍처, 테스트 작성 |
| **3. 프론트엔드 구현** | Next.js, 인증 UI, 칸반 보드 | 60분 | React 컴포넌트, App Router |
| **4. 멀티 에이전트 협업** | 서브에이전트 활용, 코드 리뷰 | 60분 | 에이전트 협업, 코드 리뷰 |
| **5. DevOps & QA** | Docker, CI/CD, 통합 테스트 | 30분 | 배포 파이프라인, 최적화 |

**총 예상 시간**: 약 4시간

## 🔍 검증 스크립트 이해하기

### 검증 스크립트란?

각 Phase 완료 후 프로젝트가 올바르게 생성되었는지 확인하는 도구입니다.

### 선택적 사용

**검증 스크립트는 필수가 아닙니다.** 다음 경우에 유용합니다:

✅ **사용하면 좋은 경우**:
- Phase가 제대로 완성되었는지 확인하고 싶을 때
- 다음 Phase로 진행하기 전 안심하고 싶을 때
- 문제를 조기에 발견하고 싶을 때
- 학습 목적으로 체크리스트를 보고 싶을 때

❌ **사용하지 않아도 되는 경우**:
- Claude와 직접 소통하며 확인할 수 있을 때
- 프로젝트가 정상 동작하는 것을 확인했을 때
- 빠르게 진행하고 싶을 때
- 자유로운 방식으로 구현하고 싶을 때

### AI 비결정성과 검증

**중요**: Claude를 포함한 AI는 같은 프롬프트에도 매번 다른 코드를 생성할 수 있습니다.

검증 스크립트는 일반적인 패턴을 확인하지만, AI가 새로운 방식으로 구현하면 **거짓 실패**가 발생할 수 있습니다.

**예시**:
```bash
# 검증 스크립트 기대: backend/Dockerfile
# AI 생성: docker/backend.Dockerfile
# → 검증 실패하지만 프로젝트는 정상
```

**판단 기준**:
1. 프로젝트가 동작하나요? → **Yes**: 검증 실패 무시
2. 핵심 기능이 있나요? → **Yes**: 검증 실패 무시
3. 다음 Phase 진행 가능한가요? → **Yes**: 검증 실패 무시

### 검증 vs Claude 확인

| 방법 | 장점 | 단점 |
|------|------|------|
| **검증 스크립트** | 빠름, 체계적 | 거짓 실패 가능 |
| **Claude에게 질문** | 컨텍스트 이해, 유연함 | 추가 프롬프트 필요 |

**추천**: 두 가지 모두 활용하되, Claude의 판단을 더 신뢰하세요.

## 🔧 프로젝트 관리 스크립트 생성

Phase 5 완료 후, 프로젝트를 쉽게 관리할 수 있는 `run.sh` 스크립트를 생성할 수 있습니다.

### 방법 1: generate-run-script.sh 사용 (권장)

```bash
# 프로젝트 디렉토리에서 실행
cd ~/ai_works/agentic-coding/taskflow-demo

# run.sh 생성
/Users/khs/ai_works/agentic-coding/docs/tutorial-automation/generate-run-script.sh .
```

또는 상대 경로로:
```bash
../docs/tutorial-automation/generate-run-script.sh .
```

### 방법 2: Claude에게 요청

Claude Code 세션 내에서:
```
"taskflow-demo/run.sh 기준으로 프로젝트 관리 스크립트를 만들어줘"
```

### run.sh 사용법

생성된 run.sh로 프로젝트를 관리할 수 있습니다:

```bash
# 전체 서비스 시작 (Docker)
./run.sh start

# 백엔드만 시작
./run.sh backend

# 프론트엔드만 시작
./run.sh frontend

# 로그 확인
./run.sh logs

# 서비스 중지
./run.sh stop

# Docker 없이 로컬 실행
./run.sh local

# 도움말
./run.sh help
```

**제공되는 기능:**
- ✅ Docker 모드: start, stop, restart, logs, status
- ✅ 개별 서비스: backend, frontend, db
- ✅ 로컬 실행 모드: local, local-backend, local-frontend
- ✅ 테스트 & 정리: test, clean, setup

## 💡 팁 & 트릭

### 1. 프롬프트를 한 번에 하나씩

각 섹션(2.2, 2.3, 2.4 등)을 별도로 실행하지 말고, Phase 전체를 한 번에 입력하세요. Claude가 자동으로 순서대로 처리합니다.

### 2. 중간에 확인하기

Claude가 작업 중일 때:
- 생성된 파일 확인: `ls -la`
- Git 상태 확인: `git status`
- 로그 확인: `docker-compose logs`

### 3. 에러 발생 시

Claude가 에러를 만나면:
```
"위 에러를 수정해줘"
```
라고 추가로 요청하세요. Claude가 자동으로 문제를 해결합니다.

### 4. 검증 스크립트 활용 (선택적)

**검증 스크립트는 선택사항**입니다. 자가 진단 도구로 활용할 수 있습니다.

```bash
# Phase 1 완료 후 검증
bash validators/validate-phase1.sh .

# 검증 통과
✓ Phase 1 검증 완료

# 검증 실패 시
✗ .claude/agents/backend-developer.md 파일이 없습니다.
```

**검증 실패 대응**:

1. **거짓 실패 가능성 확인**:
   ```bash
   # 실제 파일이 다른 위치에 있을 수 있음
   find . -name "*backend*developer*"
   ```

2. **Claude에게 확인**:
   ```
   "지금까지 잘 되고 있어? 빠진 파일이나 문제가 있어?"
   ```

3. **누락된 경우에만 요청**:
   ```
   "Phase 1 검증에서 다음 항목이 누락되었어: [항목 나열]
   이걸 추가로 생성해줘"
   ```

**참고**: 검증 스크립트가 실패해도 프로젝트가 정상 동작하면 무시해도 됩니다. AI가 다른 방식으로 구현했을 수 있습니다.

### 5. 프롬프트 커스터마이징

phases/ 디렉토리의 마크다운 파일을 직접 수정할 수 있습니다:

```bash
# Phase 2 프롬프트 수정
vim phases/02-backend.md

# 수정 후 복사해서 Claude에 입력
cat phases/02-backend.md
```

## 🔧 문제 해결

### Q: Claude가 작업을 완료했는지 어떻게 알 수 있나요?

**A:** Claude가 다음과 같이 응답하면 완료된 것입니다:
- "모든 작업이 완료되었습니다"
- "Phase X 완료"
- 더 이상 도구를 호출하지 않음

### Q: 중간에 멈춘 것 같아요

**A:** 다음을 시도하세요:
1. 잠시 대기 (복잡한 작업은 시간이 걸림)
2. Claude에게: "계속 진행해줘"
3. 또는: "현재까지 완료된 작업을 요약해줘"

### Q: 같은 Phase를 다시 실행하고 싶어요

**A:**
1. 이전 결과를 백업:
   ```bash
   cp -r taskflow-demo taskflow-demo.backup
   ```
2. 프롬프트 다시 입력
3. Claude가 기존 파일을 덮어씁니다

### Q: 특정 섹션만 실행하고 싶어요

**A:** phases/ 파일에서 필요한 섹션만 복사해서 Claude에 입력:

```bash
# Phase 2에서 "3.2 인증 시스템"만 실행
cat phases/02-backend.md | sed -n '/## 3.2/,/## 3.3/p'
```

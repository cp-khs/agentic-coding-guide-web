# TaskFlow 튜토리얼 자동 실행 도구

> 📖 **[← 메인 가이드로 돌아가기](./README.md)** | **[수동 실행 가이드 보기 →](./MANUAL-GUIDE.md)**

Claude Code를 활용하여 TaskFlow 프로젝트 튜토리얼을 **자동으로** 실행하는 도구입니다.

**자동 실행의 장점:**
- ✅ 완전 자동화된 5개 Phase 순차 실행
- ✅ 토큰 사용량 및 비용 실시간 추적
- ✅ 체크포인트 시스템으로 안전한 재개
- ✅ 약 4시간 무인 실행 가능

## 개요

이 도구는 [onboarding-tutorial.md](../onboarding-tutorial.md)에 정의된 TaskFlow 프로젝트 튜토리얼을 **자동으로 실행**합니다.

### 주요 기능

- ✅ **완전 자동화**: 5개 Phase를 순차적으로 자동 실행
- 🎯 **타겟 디렉토리 지정**: 원하는 위치에 프로젝트 생성
- 🔍 **자동 검증**: 각 Phase 완료 후 결과 검증
- 📊 **로깅**: 전체 실행 과정 로그 기록 (프로젝트별)
- ⏯️ **세션 재개**: 중단된 지점부터 안전하게 다시 시작
- 🎚️ **부분 실행**: 특정 Phase만 선택적으로 실행
- 💰 **비용 추적**: 토큰 사용량 및 API 비용 실시간 모니터링
- 🛡️ **안전장치**: 체크포인트 시스템으로 불완전한 상태 감지

### 작동 방식

이 도구는 **Headless Mode**를 활용합니다:

```
┌─────────────────────────────────────────────────────────┐
│                  실행 흐름                                │
│                                                         │
│  1. Phase 프롬프트 파일 읽기 (phases/01-*.md)            │
│           ↓                                             │
│  2. Claude Code 실행 (claude -p "...")                  │
│           ↓                                             │
│  3. 같은 세션 유지 (--resume)                            │
│           ↓                                             │
│  4. 결과 검증 (validators/validate-*.sh)                │
│           ↓                                             │
│  5. 다음 Phase 진행                                      │
└─────────────────────────────────────────────────────────┘
```

## 사용법

### 기본 사용법

```bash
# tutorial-automation 디렉토리에서
./run-tutorial.sh [타겟_디렉토리] [옵션]
```

### 예시

#### 1. 기본 실행 (현재 디렉토리 하위에 taskflow 생성)

```bash
./run-tutorial.sh
```

결과: `./taskflow` 디렉토리에 프로젝트 생성

#### 2. 특정 디렉토리에 생성

```bash
./run-tutorial.sh ~/projects/my-taskflow
```

결과: `~/projects/my-taskflow`에 프로젝트 생성

#### 3. 특정 Phase만 실행

```bash
# Phase 2부터 4까지만 실행
./run-tutorial.sh --start-phase 2 --end-phase 4
```

#### 4. 검증 건너뛰기 (빠른 실행)

```bash
./run-tutorial.sh --skip-validation
```

#### 5. 기존 세션 재개

```bash
# 이전 실행이 중단된 경우
./run-tutorial.sh --resume abc123xyz
```

### 옵션 상세

| 옵션 | 설명 | 예시 |
|------|------|------|
| `--start-phase N` | N번 Phase부터 시작 (1-5) | `--start-phase 3` |
| `--end-phase N` | N번 Phase에서 종료 (1-5) | `--end-phase 4` |
| `--skip-validation` | 검증 스크립트 건너뛰기 | `--skip-validation` |
| `--resume SESSION_ID` | 기존 세션 재개 | `--resume abc123xyz` |
| `--help` | 도움말 표시 | `--help` |

### 권한 설정

스크립트는 자동으로 `bypassPermissions` 모드로 실행되어 권한 프롬프트 없이 진행됩니다.

- 스크립트가 자동으로 `TARGET_DIR/.claude/settings.local.json` 생성
- `--permission-mode bypassPermissions` 플래그 자동 적용
- ⚠️ **주의**: 격리된 개발 환경에서만 사용하세요

**내용:**

- .dockerignore 설정
- GitHub Actions CI/CD
- 통합 테스트
- 헬스체크 설정

**검증 항목:**
- **파일 구조**: backend/Dockerfile 또는 docker/backend.Dockerfile (유연한 경로 지원)
- **멀티스테이지 빌드**: FROM ... AS 패턴 확인
- **.dockerignore**: 백엔드/프론트엔드 설정
- **CI/CD**: .github/workflows/ 워크플로우 파일
- **헬스체크**: docker-compose.yml에 healthcheck 설정

**기능 검증** (선택적):
```bash
# 자동 실행 시에는 기본적으로 파일 구조만 검증
# 수동으로 기능 검증을 실행하려면:
bash validators/validate-phase5.sh . --functional

# 실제 Docker 빌드 테스트 (1-5분 소요)
# - docker compose build backend
# - docker compose build frontend
# - 서비스 시작 테스트
```

**참고**: Phase 5는 AI가 다양한 방식으로 구현할 수 있습니다 (docker/ 디렉토리 vs 각 서비스 디렉토리, Named volume vs 익명 볼륨 등). 검증 스크립트는 여러 패턴을 허용하도록 설계되었습니다.

## 고급 사용법

### 1. 부분 실행 전략

```bash
# Phase 1만 실행 (초기 구조 확인)
./run-tutorial.sh --end-phase 1

# 결과 확인 후 나머지 실행
./run-tutorial.sh --start-phase 2
```

### 2. 실패 시 재시도

```bash
# Phase 3에서 실패한 경우
# 1. 문제 해결 후
# 2. Phase 3부터 다시 실행
./run-tutorial.sh --start-phase 3
```

### 3. 세션 ID 활용

```bash
# 첫 실행
./run-tutorial.sh | tee output.log

# 로그에서 세션 ID 확인
grep "세션 ID:" output.log

# 또는 자동 저장된 파일 확인
cat taskflow/logs/last_session_id.txt

# 세션 재개
./run-tutorial.sh --resume $(cat taskflow/logs/last_session_id.txt)
```

### 4. 병렬 프로젝트 생성

```bash
# 여러 변형 프로젝트 동시 생성
./run-tutorial.sh ~/projects/taskflow-v1 &
./run-tutorial.sh ~/projects/taskflow-v2 &
wait
```

### 5. 커스텀 Phase 프롬프트

```bash
# phases/ 디렉토리의 프롬프트 수정 후 실행
vim phases/02-backend.md
./run-tutorial.sh --start-phase 2 --end-phase 2
```

### 6. 비용 모니터링

```bash
# 실시간 비용 추적
tail -f taskflow/logs/tutorial-*.log | grep -E "토큰|비용"

# 최종 비용 확인
grep "총 비용" taskflow/logs/tutorial-*.log

# Phase별 비용 분석
grep "Phase.*비용" taskflow/logs/tutorial-*.log
```

### 7. 안전한 중단 및 복구

```bash
# Phase 완료 상태 확인
cat taskflow/.tutorial_checkpoints

# 불완전한 Phase 감지
ls taskflow/.phase_*_started 2>/dev/null

# Git 히스토리 확인
cd taskflow && git log --oneline --graph

# 안전한 시점으로 복원
cd taskflow && git reset --hard <commit-hash>
```

### 8. 검증 스크립트 활용

**자동 실행 시**:
```bash
# 기본: 검증 자동 실행
./run-tutorial.sh

# 검증 건너뛰기 (빠른 실행)
./run-tutorial.sh --skip-validation

# Phase 5 기능 검증 추가 (수동)
cd taskflow
bash ../tutorial-automation/validators/validate-phase5.sh . --functional
```

**검증 실패 대응**:
```bash
# 1. 상세 로그 확인
bash validators/validate-phase1.sh . 2>&1 | tee validation.log

# 2. 실제 파일 확인
ls -la backend/
find . -name "*Dockerfile*"

# 3. 프로젝트가 동작하는지 확인
docker compose build
docker compose up -d

# 4. 동작하면 검증 무시하고 계속 진행
./run-tutorial.sh --start-phase 2 --skip-validation
```

**AI 비결정성 이해**:

Claude를 포함한 AI는 같은 프롬프트에도 매번 다른 코드를 생성할 수 있습니다.
이는 버그가 아니라 AI의 **창의성**이자 **비결정성**입니다.

검증 스크립트는 다음을 허용합니다:
- ✅ 다양한 디렉토리 구조 (`backend/Dockerfile`, `docker/backend.Dockerfile`)
- ✅ 다양한 파일명 패턴
- ✅ 키워드 기반 기능 확인 (특정 파일명이 아닌 내용 검색)

하지만 **새로운 예측하지 못한 패턴**은 검증 실패를 유발할 수 있습니다.

**판단 기준**:
- 검증 실패 → 프로젝트 동작 안 함: **실제 문제**
- 검증 실패 → 프로젝트 정상 동작: **거짓 실패** (무시 가능)

### 문제 2: 권한 거부 (Permission denied)

**증상:**
```
bash: ./run-tutorial.sh: Permission denied
```

**해결:**
```bash
chmod +x run-tutorial.sh
chmod +x validators/*.sh
```

---

### 문제 3: Phase 실행 타임아웃

**증상:**
```
ERROR: Phase 2 실행 실패
```

**원인:** 복잡한 Phase (특히 Phase 2, 3)는 시간이 오래 걸릴 수 있음

**해결:**
1. `run-tutorial.sh`의 `CLAUDE_TIMEOUT` 값 증가 (기본값: 1200초)
   ```bash
   # run-tutorial.sh 편집
   CLAUDE_TIMEOUT=2400  # 40분으로 증가
   ```

2. 또는 Phase별로 분리 실행
   ```bash
   ./run-tutorial.sh --start-phase 2 --end-phase 2
   ```

### 문제 4: jq/bc 없음 경고

**증상:**
```
WARN: jq가 설치되어 있지 않습니다. JSON 파싱 및 토큰 추적을 위해 설치를 권장합니다.
WARN: bc가 설치되어 있지 않습니다. 비용 계산을 위해 설치를 권장합니다.
```

**영향:**
- **jq 없음**: 세션 ID/토큰 자동 추출 불가
- **bc 없음**: 비용 계산 불가

**해결:**
```bash
# macOS
brew install jq bc

# Linux
sudo apt-get install jq bc
```

```
ERROR: 디렉토리가 이미 존재합니다: ./taskflow
```

  ```bash
  cat ./taskflow/logs/last_session_id.txt
  ./run-tutorial.sh --resume <SESSION_ID>
  ```

### 문제 6: Phase가 불완전한 상태

**증상:**
```
⚠️  Phase 2가 이전에 시작되었지만 완료되지 않았습니다!
부분적으로 변경된 파일이 있을 수 있습니다.
```

**원인:** Phase 실행 중 중단됨 (Ctrl+C, 에러 등)

**해결 방법:**

**옵션 1: Git으로 이전 상태 복원 (안전)**
```bash
cd taskflow
git log --oneline  # Phase 1 커밋 찾기
git reset --hard <commit-hash>  # 안전한 상태로 복원
cd ..
./run-tutorial.sh --start-phase 2
```

**옵션 2: 마커 파일 제거 후 계속 (주의)**
```bash
rm taskflow/.phase_2_started
./run-tutorial.sh --start-phase 2
# ⚠️ 파일 충돌 가능성 있음
```

**옵션 3: 처음부터 다시**
```bash
rm -rf taskflow
./run-tutorial.sh
```

---

### 문제 7: 검증 실패

**증상:**
```
WARN: Phase 2 검증 실패 (계속 진행)
```

**의미:** Phase가 완료되었지만 일부 파일/구조가 예상과 다름

**대응:**
1. 로그 확인: `cat taskflow/logs/tutorial-*.log`
2. 수동으로 확인: `cd taskflow && ls -la`
3. 필요시 해당 Phase만 재실행

### 문제 8: 토큰 사용량이 0으로 표시됨

**증상:**
```
WARN: 토큰 사용량 정보를 가져올 수 없었습니다.
```

**원인:**
- jq가 설치되지 않음
- Claude CLI의 JSON 출력 형식 변경
- --output-format json이 지원되지 않음

**해결:**
1. jq 설치: `brew install jq`
2. Claude Code 버전 확인: `claude --version` (v0.5.0 이상 권장)

## FAQ

### Q1. 전체 실행 시간은?

**A:** 약 4시간 예상 (Phase별 시간)
- Phase 1: 30분
- Phase 2: 60분
- Phase 3: 60분
- Phase 4: 60분
- Phase 5: 30분

실제 시간은 시스템 성능과 Claude Code 응답 속도에 따라 다름

---

### Q2. 중간에 멈출 수 있나?

**A:** 네, `Ctrl+C`로 중단 가능하지만 주의 필요
- ✅ 세션 ID는 `TARGET_DIR/logs/last_session_id.txt`에 저장됨
- ✅ `--resume` 또는 `--start-phase` 옵션으로 재개 가능
- ⚠️ **주의**: Phase 중간에 중단하면 불완전한 상태가 될 수 있음
- 💡 **권장**: Phase 완료 후 로그를 확인하고 안전한 시점에 중단

**재개 방법:**
```bash
# 방법 1: Phase 번호로 재개 (권장)
./run-tutorial.sh --start-phase 3

# 방법 2: 세션 ID로 재개
cat taskflow/logs/last_session_id.txt
./run-tutorial.sh --resume <SESSION_ID>
```

---

### Q3. 여러 번 실행해도 되나?

**A:** 주의해서 실행 필요

**새로 시작 (권장):**
```bash
rm -rf taskflow
./run-tutorial.sh
```

**재개 (기존 작업 계속):**
```bash
# 완료된 Phase 확인
grep "completed" taskflow/.tutorial_checkpoints

# 다음 Phase부터 실행
./run-tutorial.sh --start-phase <다음번호>
```

**⚠️ 경고:**
- 같은 디렉토리에 중복 실행하면 에러 발생
- Phase가 불완전한 상태면 경고 메시지 표시
- 파일 충돌 가능성 있음

---

### Q4. Phase 순서를 바꿔도 되나?

**A:** 권장하지 않음
- Phase는 순차적 의존 관계가 있음
- Phase 1 없이 Phase 2 실행 시 오류 발생 가능

### Q5. Claude Code API 키 설정은?

**A:** 사전에 설정 필요
```bash
claude login
```

### Q6. 프롬프트를 수정하고 싶다면?

### Q8. 로그는 어디에 저장되나?

**A:** `TARGET_DIR/logs/` 디렉토리 (프로젝트별 저장)

**예시:**
```bash
taskflow/logs/
├── tutorial-20260213-191425.log  # 실행 로그
└── last_session_id.txt            # 세션 ID
```

**로그 확인:**
```bash
# 실시간 모니터링
tail -f taskflow/logs/tutorial-*.log

# 최근 로그
tail -50 taskflow/logs/tutorial-*.log

# 에러 검색
grep ERROR taskflow/logs/tutorial-*.log
```

### Q11. 토큰 비용은 얼마나 나오나?

**A:** 프로젝트 크기와 Phase에 따라 다름

**예상 비용 (Sonnet 4.5 기준):**
- Phase 1: $0.3-0.5
- Phase 2: $0.5-1.0
- Phase 3: $0.5-1.0
- Phase 4: $0.4-0.8
- Phase 5: $0.2-0.4
- **전체 예상**: $2-4 USD

**비용 확인:**
```bash
# 실시간 확인 (로그에서)
tail -f taskflow/logs/tutorial-*.log | grep "비용"

# 최종 요약
grep "총 비용" taskflow/logs/tutorial-*.log
```

**비용 절감 팁:**
- `--skip-validation`: 검증 건너뛰기
- 특정 Phase만 실행: `--start-phase N --end-phase N`
- 불필요한 Phase 제외

---

### Q12. 체크포인트 파일은 무엇인가?

**A:** Phase 완료 상태를 추적하는 안전장치

**파일 위치:**
- `taskflow/.tutorial_checkpoints`: 완료된 Phase 목록
- `taskflow/.phase_N_started`: Phase N 실행 중 마커

**내용 확인:**
```bash
# 완료된 Phase 확인
cat taskflow/.tutorial_checkpoints

# 예시 출력:
# phase_1_completed=1707828960
# phase_2_completed=1707830120
```

**활용:**
- 이미 완료된 Phase는 자동 건너뛰기
- 불완전한 Phase 감지 및 경고
- 안전한 재개 지원

---

**A:**

```bash
# Phase 3이 실패한 경우
./run-tutorial.sh --start-phase 3 --end-phase 3
```

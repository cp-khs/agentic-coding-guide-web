# Phase 2: 백엔드 구현 (60분)

## 3.1 데이터베이스 모델 및 마이그레이션

**중요: 다음 디렉토리 구조를 따라서 생성해줘.**

백엔드 데이터베이스 모델을 `backend/app/models/` 디렉토리에 구현해줘.

### 필요한 모델 파일:

- `backend/app/models/__init__.py` (모델들을 export)
- `backend/app/models/user.py` - User 모델
- `backend/app/models/project.py` - Project, ProjectMember 모델
- `backend/app/models/task.py` - Task, Comment 모델

### 모델 요구사항:

- **User**: 사용자 정보 (이메일, 비밀번호, 이름)
- **Project**: 프로젝트 정보와 소유자 관계
- **ProjectMember**: 프로젝트 멤버십과 역할 (owner/admin/member)
- **Task**: 태스크 정보 (제목, 설명, 상태, 우선순위, 담당자)
  * status는 todo/in_progress/done으로 해줘
- **Comment**: 태스크에 대한 댓글

Alembic으로 마이그레이션 설정도 함께 해줘 (`backend/alembic/`).

## 3.2 인증 시스템

JWT 기반 인증 시스템을 구현해줘.

필요한 기능:
- 회원가입 (POST /api/auth/register)
- 로그인 (POST /api/auth/login) - JWT 토큰 발급
- 현재 사용자 정보 조회 (GET /api/auth/me)
- 비밀번호 해싱 처리
- 인증이 필요한 엔드포인트에서 사용할 의존성 함수

테스트도 함께 작성해줘.

---

## 3.3 핵심 API 엔드포인트

프로젝트와 태스크 관리를 위한 REST API를 구현해줘.

프로젝트 API:
- CRUD (생성, 목록, 상세, 수정, 삭제)
- 멤버 추가 기능

태스크 API:
- CRUD (생성, 목록, 상세, 수정, 삭제)
- 상태 변경 (PATCH)
- 목록 조회 시 필터링과 정렬 지원

댓글 API:
- 댓글 작성 및 목록 조회

모든 엔드포인트는 인증 필요, 프로젝트 멤버만 접근 가능하게 해줘.
각 API는 Pydantic 스키마, 서비스 레이어, 라우터, 테스트를 포함해야 해.

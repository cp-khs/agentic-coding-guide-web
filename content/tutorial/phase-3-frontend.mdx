## 1.3 칸반 보드 UI

이것이 프론트엔드의 핵심 기능이다. 복잡한 UI이므로 Plan Mode를 먼저 사용하는 것이 좋다.

```
# Plan Mode에서 먼저 계획 확인

칸반 보드 UI의 구현 계획을 세워줘.
Todo / In Progress / Done 세 컬럼, 태스크 카드 드래그 앤 드롭,
태스크 상세 모달, 새 태스크 생성 기능이 필요해.
컴포넌트 분리 전략과 상태 관리 방식을 설명해줘.
```

계획을 확인한 후 실행을 지시한다.

```
# 일반 모드로 전환

방금 세운 계획대로 칸반 보드를 구현해줘.

페이지:
- /projects - 프로젝트 목록
- /projects/[id] - 칸반 보드 (핵심 페이지)

컴포넌트:
- KanbanBoard - 보드 전체 레이아웃
- KanbanColumn - 상태별 컬럼 (Todo, In Progress, Done)
- TaskCard - 태스크 카드 (제목, 우선순위 뱃지, 담당자 아바타)
- TaskDetailModal - 태스크 상세 보기/수정 모달
- CreateTaskForm - 새 태스크 생성 폼
- CommentSection - 댓글 목록 및 작성

드래그 앤 드롭은 HTML5 Drag and Drop API나
간단한 구현으로 해줘 (외부 라이브러리 최소화).
```

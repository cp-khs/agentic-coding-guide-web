## 2.3 프로젝트 스캐폴딩

**중요: 다음 디렉토리 구조를 정확히 따라서 생성해줘.**

이 프로젝트의 전체 디렉토리 구조를 만들어줘.
백엔드와 프론트엔드를 분리하고, Docker Compose로 통합 개발 환경을 구성해줘.

### 필수 디렉토리 구조:

```
project-root/
├── backend/
│   ├── app/
│   │   ├── __init__.py
│   │   ├── main.py
│   │   ├── api/          # API 라우터
│   │   ├── models/       # SQLAlchemy 모델
│   │   ├── schemas/      # Pydantic 스키마
│   │   ├── services/     # 비즈니스 로직
│   │   └── core/         # 설정, 보안, DB
│   ├── tests/
│   ├── alembic/
│   ├── pyproject.toml
│   └── alembic.ini
│
├── frontend/
│   ├── app/              # Next.js App Router
│   ├── components/
│   ├── lib/
│   ├── public/
│   ├── package.json
│   ├── tsconfig.json
│   └── next.config.js
│
├── docker/
│   ├── backend.Dockerfile
│   └── frontend.Dockerfile
│
├── docker-compose.yml
├── .env.example
├── .gitignore
└── README.md
```

### 필수 파일들:

1. **backend/** - FastAPI 앱 (위 구조대로)
2. **frontend/** - Next.js 앱
3. **docker/** - Dockerfile들 (backend.Dockerfile, frontend.Dockerfile)
4. **docker-compose.yml** - 백엔드, 프론트엔드, PostgreSQL 서비스 (db, backend, frontend)
5. **.env.example** - 환경 변수 템플릿
6. **각 서비스의 기본 설정 파일** (pyproject.toml, package.json 등)

## 2.4 서브에이전트 정의

**중요: 다음 경로와 파일명을 정확히 따라서 생성해줘.**

`.claude/agents/` 디렉토리를 만들고 다음 서브에이전트를 정의해줘.

**필수 요구사항:**
- 각 파일은 반드시 맨 앞에 YAML frontmatter(name, description, tools)를 포함해야 해
- Frontmatter가 없으면 서브에이전트로 인식되지 않아

**Frontmatter 예시:**
```markdown
---
name: Backend Developer
description: FastAPI 백엔드 개발 전문가
tools:
  - Read
  - Write
  - Edit
  - Bash
  - Glob
  - Grep
---

# Backend Developer

...
```

### 생성할 서브에이전트 파일:

1. **.claude/agents/backend-developer.md**
   - name: Backend Developer
   - FastAPI 백엔드 전문
   - tools: Read, Write, Edit, Bash, Glob, Grep

2. **.claude/agents/frontend-developer.md**
   - name: Frontend Developer
   - Next.js 프론트엔드 전문
   - tools: Read, Write, Edit, Bash, Glob, Grep

3. **.claude/agents/qa-engineer.md**
   - name: QA Engineer
   - 테스트 작성 및 품질 검증
   - tools: Read, Write, Edit, Bash, Glob, Grep

4. **.claude/agents/code-reviewer.md**
   - name: Code Reviewer
   - 코드 리뷰 (읽기 전용)
   - tools: Read, Grep, Glob

5. **.claude/agents/devops-engineer.md**
   - name: DevOps Engineer
   - Docker, CI/CD 설정
   - tools: Read, Write, Edit, Bash, Glob, Grep

각 에이전트의 역할, 규칙, 출력 형식을 상세하게 작성해줘.
